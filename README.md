## Five Principles of Object-Oriented Programming (OOP)

I have learned and practiced the five core principles of OOP in Java:

### 1. Encapsulation (کپسوله‌سازی)
- Hiding internal data and exposing only necessary methods.
- Using `private` fields and `public` getters and setters.

### 2. Inheritance (ارث‌بری)
- Creating a new class based on an existing class.
- Using `extends` to inherit properties and methods.
- Using `super` to access parent class members.

### 3. Polymorphism (پلی‌مورفیسم)
- One name, many behaviors.
- Method overloading (compile-time) and overriding (runtime).
- Using a parent reference to refer to child objects.

### 4. Abstraction (انتزاع)
- Hiding implementation details and showing only functionality.
- Using `abstract` classes and methods.
- Cannot instantiate abstract classes.

### 5. Interface (واسط)
- A contract that classes must follow.
- Using `interface` to define methods without implementation.
- Classes use `implements` to implement interfaces.
- Supports multiple inheritance of type.
